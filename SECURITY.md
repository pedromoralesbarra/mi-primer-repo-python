# Security Policy

## Supported Versions

The `main` branch receives security updates.

## Reporting a Vulnerability

Do not publish secrets, tokens, or personal data in public issues.

If you find a vulnerability, report it privately to the repository owner and include:

- affected repository and branch
- steps to reproduce
- expected and actual behavior
- any logs without credentials or personal data

## Operational Notes

- Keep `.env`, virtual environments, and generated files out of Git.
- Rotate any token immediately if it is accidentally committed.
